## 1. サービス概要（※最後に書いてOK）

- このサービスはどんなサービスか（3行以内）
日常の些細な善行と悪行を「徳（ポイント）」として可視化し、現在の徳分に基づいた「来世の転生先」をリアルタイムで予測するエンタメ・徳積みカウンターです。

- 誰の、どんな課題を、どう解決するか
SNSの数字やネット上の反応ばかりを気にし、目の前の現実世界での行動が疎かになっている現代人の課題を、「現実の善行をゲーム化する」ことで解決し、オフラインの生活にやりがいを与えます。

「徳」という言葉が使われているものの、それはゲーム内の積み上げブロックの要素として扱われており、
現実の行動とつなげ、振り返る仕組みにはなっていません。

## 8. このアプリの懸念点とその対策
※最低1つは必ず書いてください。

このアプリが**うまくいかない可能性**と、
それに対して**どう向き合おうとしているか**を書いてください。

### 懸念点①

- 何が問題になりそうか：
ユーザーが「嘘の申告」をして徳を無限に稼げてしまう。

- なぜそう思うか：
自己申告制である以上、システム的に嘘を完全に防ぐことは難しいため。

- そのために考えている対策：
本アプリは「自分の良心との対話」を楽しむツールであることを明確にします。
また、あえて「嘘をついて徳を稼ぐ」という行動を選ぶと称号が固定されたり、ポイントがリセットされるなど、ユーモアのある制裁を検討します。

## 9. 今後の展開・発展の方向性

このアプリは、
**「投稿して終わり」にならないために**
どんな広がりが考えられますか？

### 観点（すべて答えなくてOK）

- ユーザーはなぜ使い続ける？
徳や業の記録が蓄積され、ポイントや称号、来世予測が変化していくことで、ゲームのような成長体験が得られるためです。
また、日常の小さな行動を振り返る習慣が生まれることで、継続的に利用する動機になります。
- 続けることで何が変わる？
ユーザーが日常生活の中で善行を意識するようになり、周囲の状況や他人への配慮に気づきやすくなると考えています。
また、スマートフォン中心の生活から、現実世界での行動を意識するきっかけになる可能性があります。
- データが溜まると何ができる？
徳や業の履歴データを分析することで、ユーザー自身の行動傾向を振り返ることができます。
例えば「どんな行動で徳を積んでいるのか」「どの時間帯に徳が増えやすいか」などを可視化することで、ユーザーが自分の行動パターンを理解することができます。
- 他の人と関わる余地はある？
将来的には、匿名で徳の記録を共有したり、他のユーザーの善行を見ることができる機能を検討しています。
これにより、自分以外のユーザーの行動がモチベーションとなり、善行を継続するきっかけになると考えています。
- 運用・仕組みで価値を出せないか？
ユーザー全体の徳の総量を可視化し、「今日の総徳量」や「週間徳ランキング」などを表示することで、コミュニティとしての楽しさを生み出す仕組みを検討しています。

### 今考えている発展案

- 機能追加の方向性：
日本全体のユーザーが積んだ「総徳分」を可視化し、全体の傾向をユーモアを交えて表示する。

- UI・体験の改善：
徳の状態に応じて画面演出が変化し、モチベーションを維持できるようにする。

- 機能以外の工夫：
街中の清掃活動など、現実のオフライン活動と連動できる仕組みの検討。

### 10-2. 技術選定の理由

- なぜこの技術を使うのか
Next.jsで徳入力後にポイントや来世予測が即座に変わる、リッチで高速なユーザー体験を実現するため。
Railsでユーザーごとの徳・業の履歴管理や、称号ロジックを柔軟に実装するため。

- 今回チャレンジしたい点
ポイントや称号、来世予測を「条件に応じて動的に変化させる」ロジック設計と実装。

- 不安な点
称号や来世予測のパターンを飽きさせないほど十分に用意するためのコンテンツ制作コスト。
